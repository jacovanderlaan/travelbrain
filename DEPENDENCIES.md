# Dependencies — travel-curation

What each package is for, per component. Versions are floors (`>=`), pinned loosely;
tested with the "tested" column as of 2026-06-28.

## Python

| Package | Used by | Purpose | Tested |
|---------|---------|---------|--------|
| `duckdb` | brain, mcp-server, pipeline, site export | The analytical store — the whole "brain" (schema, scoring, content) is a single DuckDB file. Pure-Python wheel, no server. | 1.4.4 |

| `Pillow` | (manual, header prep) | Resize/orient own photos to web size before adding as headers. Not imported by the pipeline; used ad-hoc. | 12.2.0 |

## Own photos + location matching

Headers now use **Jaco's own Sicily photos** (April 2022 trip) from `D:/media/photos/private/2022/2022-04/`, resized to ~1600px and committed under `site/public/photos/`. Credit: Jaco van der Laan.

`pipeline/match_photo_location.py` solves the geo gap: most photos lack GPS, but
`photos.duckdb.timeline_points` (176k Google-Timeline GPS points, 2013-2026) lets us
match a photo's capture time → nearest GPS point → nearest GeoNames city. Verified on
the Sicily trip: 1,015 photos placed (265 Siracusa, 188 Palermo, …). Reads PHOTOS_DB
(default `D:/data/databases/photos.duckdb`, read-only). See memories
`photos-duckdb-geocoding-gap`, `sicily-2022-trip-real`, `locations-data-source`.

Install per component:

```bash
pip install -r brain/requirements.txt        # duckdb  (if present; brain only needs duckdb)
pip install -r mcp-server/requirements.txt    # mcp, duckdb
pip install -r pipeline/requirements.txt      # duckdb, anthropic
```

Standard-library only (no install): `os, re, json, sys, argparse, hashlib, asyncio,
pathlib, dataclasses, datetime, typing`.

## Rebuild-order gotcha

`brain/build_db.py --reset` rebuilds the brain from schema + seed + research SQL.
It does NOT recreate **generated** rows: the drafted page (`dim_content`) and
approved images (`dim_image`). After a `--reset`, re-run the pipeline to restore them:

```bash
python brain/build_db.py --reset
python pipeline/draft_page.py --destination Palermo --draft-file pipeline/palermo_draft.json
python pipeline/fetch_images.py --all     # + --approve <sk> per venue (needs an image key)
cd site && npm run build
```
